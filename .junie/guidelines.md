# Инструкции для Junie (AI-ассистента)

Данный документ содержит обязательные правила и стандарты разработки для этого проекта.

## 1. Язык комментариев
- **Все комментарии в коде должны быть на русском языке.**
- Это касается JSDoc, inline-комментариев и описаний интерфейсов.
- Код (переменные, функции, классы) остается на английском языке согласно стандартам индустрии.

## 2. Документирование кода
Необходимо подробно документировать:
- **Основную логику:** пояснение "почему" сделано именно так, а не только "что" делает код.
- **Нестандартные решения:** любые хаки, оптимизации или специфичные для проекта подходы.
- **Нюансы:** возможные побочные эффекты, зависимости от внешних состояний или ограничения реализации.

## 3. Подход TDD (Test-Driven Development)
Разработка любой новой функциональности должна следовать циклу TDD:
1. **Red:** Написание теста, который описывает ожидаемое поведение и падает.
2. **Green:** Написание минимально необходимого кода для того, чтобы тест прошел.
3. **Refactor:** Улучшение кода при сохранении работоспособности тестов.

### Инструментарий для тестов:
- **Backend:** Vitest или Jest.
- **Frontend:** Vitest + Testing Library.

## 4. Стилистика и структура
- Соблюдать единообразие с существующим кодом.
- Использовать функциональный подход в React-компонентах.
- Предпочитать типизацию TypeScript вместо any.
- **Запрещено создавать или перемещать файлы в одинаковые подпапки** (например, `client/client` или `server/server`).

## 5. Использование инструментов и команд
- **Менеджер пакетов:** Обязательно использовать `pnpm`.
- **Работа с терминалом:** 
  - **Категорически запрещено использовать `ls -R`** или аналогичные команды рекурсивного вывода всех файлов (например, `Get-ChildItem -Recurse`). Это приводит к переполнению контекста и зависанию чата из-за большого количества файлов.
  - Для изучения структуры использовать `Get-ChildItem` с ограничением глубины (например, `-Depth 2`) или специализированные инструменты поиска.
